Flagship Product Streamlit / FastAPI / Render / Android

Matrika Academy

As CEO of Matrika Yoga, I shaped this app to bring the whole academy journey into one calm flow: discovery, account creation, admissions, live class access, replay follow-up, payments, contact, and protected admin review.

  • 11 pages: Dashboard, Account, Programs, Schedule, Admissions, Live Studio, Certification, Kids Studio, Payments, Contact, and Admin.
  • 10 persisted data tables covering bookings, attendance, certification, kids enquiries, payments, contact, user accounts, password resets, reply logs, and Razorpay links.
  • Dual storage model with Google Sheets for cloud persistence and local CSV fallback for development.
  • Automatic email replies, password reset verification, WhatsApp support paths, live Zoom joining, replay requests, and account-linked payment flows.
  • SEO wrapper with landing page, JSON-LD, `robots.txt`, `sitemap.xml`, and `/app` routing for a cleaner public domain experience.

Delivery Web + SEO + Android

The academy can be served from a public landing page, a Streamlit app shell, and an Android WebView release URL.

Page system

Every core academy path is covered.

The app is structured as a complete learner product, not just a form hub. Visitors can browse programs and schedules, then move into account-backed admissions, live studio, certification, kids enquiries, payments, and support.

  • Programs include Garbhasanskara Flow, Prenatal + Postnatal Care, Kids Yoga Studio, Teacher Certification, and Everyday Wellness.
  • Schedule guidance includes morning, afternoon, evening, weekend, and flexible time-period options.
  • Live Studio combines live links, replay support, and attendance in a single class experience.

Payments + support

Modern checkout with fallback paths.

The payments page supports both a premium hosted flow and the academy’s older manual process, so learners can pay in the way that feels easiest.

  • Hosted Razorpay payment links can be generated per learner and tracked in the admin data.
  • Manual UPI fallback stays available with provider shortcuts for Google Pay, PhonePe, Paytm, BHIM, Amazon Pay, and other UPI apps.
  • Payment proofs, transaction references, and follow-up confirmations remain tied to the signed-in account.

Growth + deployment

Built to be shared publicly.

The project is ready for Render or Streamlit Community Cloud, and it also includes an Android wrapper for a lightweight mobile app release.

  • `seo_server.py` serves a crawlable public landing page at `/`, then routes the Streamlit app under `/app`.
  • The public domain configuration points at `matrikayogaacademy.com` with SEO metadata, FAQ schema, favicon, and sitemap support.
  • The Android wrapper loads the local Streamlit server in debug and a production HTTPS URL in release builds.
Detailed page map

Dashboard gives the academy overview, Account handles sign up and login, Programs and Schedule explain the offering, Admissions captures enrollment requests, Live Studio manages live class access and attendance, Certification and Kids Studio run focused forms, Payments handles Razorpay and UPI flows, Contact manages support, and Admin is a password-protected operations surface.

Data model and integrations

Saved datasets cover admissions bookings, attendance, certification applications, kids enquiries, payments, contact messages, user accounts, password reset requests, automatic reply logs, and Razorpay links. Integrations include Google Sheets, SMTP email delivery, Razorpay APIs, WhatsApp sharing, Zoom joining, and account-linked UPI shortcuts.

Hosting and admin workflow

Render uses `uvicorn seo_server:app` with `APP_BASE_PATH=/app`, while Streamlit Community Cloud uses `app.py` plus secrets for Google Sheets, SMTP, admin password, and Razorpay. Admin review shows storage mode, row counts, recent entries, spreadsheet links when cloud persistence is active, and CSV exports for each workflow.
